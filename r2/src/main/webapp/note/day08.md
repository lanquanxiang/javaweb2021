# 1. 数据库连接池
## 1. 原生JDBC存在的问题
1. 频繁的创建和销毁Connection-->消耗时间和IO资源
2. 消耗时间-->降低数据库操作效率
3. 消耗IO资源-->没有对链接的创建进行限制（无限制创建）-->数据库崩溃
## 2. 数据库连接池
1. 在程序启动时初始化一批链接放在连接池（集合）中【如何初始化】
2. 用“申请”来代替“创建”【如何获得链接】
3. 用“归还”来代替“销毁”【如何归还链接】
【面试】简述数据库连接池的实现
